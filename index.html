Q3 Explain basic operational concepts between processor & Memory
4 Marks ▾

Key Registers ↗ Learn more

RegisterFull NameFunction
PCProgram CounterHolds address of NEXT instruction to fetch
IRInstruction RegisterHolds instruction currently being executed
MARMemory Address RegisterHolds address being accessed in memory
MDRMemory Data RegisterHolds data being read/written to memory

Fetch-Execute Cycle

Step 1: PC → holds address of next instruction Step 2: Address sent to Memory → instruction FETCHED → placed in IR Step 3: PC incremented to point to NEXT instruction (PC = PC + 4) Step 4: Control unit DECODES instruction in IR Step 5: ALU EXECUTES the operation Step 6: Result stored back in memory or register

Two Memory Operations

  • READ: Processor sends address + READ signal → Memory returns data copy. Original stays unchanged.
  • WRITE: Processor sends address + data + WRITE signal → Memory overwrites old data.

Q4 Elaborate Single Bus Architecture in detail
4 Marks ▾

What is a Bus?

A bus is a group of wires that connects all parts of a computer and carries data between them. The single bus connects the ALU and ALL registers via one common internal path.

Diagram

┌────────────────── SINGLE INTERNAL BUS ──────────────────┐ │ │ PC MAR MDR IR R0 R1 ... R(n-1) Y Z TEMP │ │ │ │ Address Data │ Bus Bus ← connects to external memory │ └──► ALU (with MUX selecting Y or constant 4)

Key Components

ComponentRole
MAROutput connects to external address bus → points to memory location
MDRTwo inputs (internal bus / memory bus), two outputs. Bridges processor ↔ memory
Y, Z, TEMPTemporary registers used during multi-step instruction execution
MUXSelects either register Y or constant 4 as ALU input A

Register Transfer (How data moves)

Each register has two control signals: Riin (load from bus) and Riout (place on bus). To transfer R1 → R4: set R1out=1 (puts R1 on bus), then set R4in=1 (loads bus into R4).

⚡ Advantage: Simple design, low cost
⚠️ Disadvantage: SLOW — only ONE transfer per clock cycle

Q7 Elaborate the working of Memory Addresses and its Operations
4 Marks ▾

Memory Organization

  • Memory = millions of cells, each stores 1 bit (0 or 1)
  • Cells grouped into words (16, 32, or 64 bits typically)
  • Each word has a unique address starting from 0
  • A unit of 8 bits = 1 byte
Address Content 0000 → [ word 0 ] 0001 → [ word 1 ] 0002 → [ word 2 ] ... FFFF → [ last word ]

READ Operation

  • Processor sends address + READ signal to memory
  • Memory retrieves and sends a copy to processor
  • Original data in memory stays unchanged ✅

WRITE Operation

  • Processor sends address + data + WRITE signal
  • Memory writes new data to that location
  • Old data is overwritten and lost ⚠️

Q9 Write short note on Stack Memory and its Operations
4 Marks ▾

What is a Stack? ↗ GeeksforGeeks

A stack is a special memory structure that follows the LIFO (Last In, First Out) principle. Think of a pile of plates — you always add and remove from the TOP!

Key Terms

  • TOP — where items are added/removed
  • BOTTOM — the very first item placed
  • SP (Stack Pointer) — register that always points to current TOP

PUSH Operation (Adding to stack)

Subtract SP, SP, #4 ← SP moves DOWN (stack grows downward) Store Rj, (SP) ← new item placed at TOP

POP Operation (Removing from stack)

Load Rj, (SP) ← take TOP item into register Rj Add SP, SP, #4 ← SP moves UP (new TOP revealed)

Stack Diagram

SP → [ -28 ] ← TOP (most recently added) [ 17 ] [ 739 ] BOTTOM → [ 43 ] ← BOTTOM (first added, last removed)
Uses of Stack: Return addresses for subroutines · Parameter passing · Zero-address instruction evaluation (postfix expressions)

What is a Subroutine? ↗ GeeksforGeeks

A subroutine is a block of instructions that performs a specific task and can be called multiple times from different places in a program. Also called: function, procedure, or method.

Why Use Subroutines?

  • Saves memory — write code once, use many times
  • Makes programs organized and readable
  • Reduces repetition and errors

Two Key Instructions

InstructionAction
CALLSaves current PC into Link Register → jumps to subroutine start
RETURNLoads Link Register back into PC → returns to calling program

Diagram

Memory Calling Program Memory Subroutine SUB 200 → Call SUB ──────────────► 1000 first instruction 204 ← next instruction ◄────── ... ... ... ... Return ↑ execution resumes here after Return

Subroutine Nesting

When a subroutine calls ANOTHER subroutine — return addresses are saved on the STACK (not just link register) to correctly track all returns.

Two Address Format

Specifies TWO address fields. Result usually overwrites one source operand. Common in commercial computers.

[ OPCODE | Destination | Source | mode ]

Three Address Format

Specifies THREE address fields. Result stored separately. Programs are shortest in size but instruction bits increase.

Q6 State advantages of Zero, One, Two and Three address instruction format
4 Marks ▾

Zero Address — Advantages

  • Reduced instruction set — simpler CPU design
  • Less decoding complexity
  • Efficient for nested/recursive operations
  • Fewer bits needed (no address field)

One Address — Advantages

  • Balance between simplicity and versatility
  • Simpler operand fetching (only one explicit operand)
  • Good code density — smaller program size vs 2/3 address
  • Implicit accumulator speeds up some operations

Two Address — Advantages

  • Allows direct operation on memory or registers
  • More flexible than one address
  • Intermediate results can be stored directly
  • Better performance — fewer memory accesses

Three Address — Advantages

  • Direct representation of complex expressions
  • Shortest program length (fewest instructions)
  • Supports parallelism — multiple operands fetched simultaneously
  • Better pipeline utilization and register allocation

Q7 Discuss limitations of Zero, One, Two and Three address instruction format
4 Marks ▾

Zero Address — Limitations

  • Heavy stack dependency causes access inefficiency
  • Frequent PUSH/POP operations hurt performance
  • Difficult to optimize — operand access is implicit
  • Harder to debug than register-based operations

One Address — Limitations

  • Accumulator bottleneck — reduces parallelism
  • More instructions needed for complex operations
  • Extra data movement between memory and accumulator

Two Address — Limitations

  • Result overwrites source operand — needs extra instructions to preserve data
  • Larger instruction size than 0/1 address
  • More decoding complexity

Three Address — Limitations

  • Largest instruction size — more memory per instruction
  • Complex instruction decoding — affects performance
  • Higher hardware cost and power consumption
  • Longer operand fetch time

Q9 Explain Multi-Bus Organization in detail
4 Marks ▾

Why Multi-Bus?

Single bus is slow (only 1 transfer per clock). Multi-bus provides multiple internal paths enabling several transfers in parallel → significantly faster!

Three-Bus Structure

Bus A ──────────────────────────────────────────► Bus B ──────────────────────────────────────────► } to ALU inputs Bus C ◄─────────────────────────────────────────── ALU output Register file has 3 PORTS: Port A → output to Bus A (source operand 1) Port B → output to Bus B (source operand 2) Port C → input from Bus C (destination result)

How it Works

  • Bus A and B carry TWO source operands to ALU simultaneously
  • ALU performs operation and sends result over Bus C
  • All three happen in the SAME clock cycle!
  • No need for temporary registers Y and Z (unlike single bus)

Comparison

FeatureSingle BusMulti-Bus
Transfers per cycle13 simultaneously
SpeedSlowMuch faster
CostLowHigher
Temp registers neededY, Z, TEMPNot needed
Extra feature: Incrementer unit increments PC by 4 without using the main ALU — freeing ALU for actual computation!

Q10 Write short note on Hardwired Control Unit
4 Marks ▾

What is Hardwired Control? ↗ GeeksforGeeks

Control signals are generated by actual hardware circuits (combinational logic) that directly interpret the instruction register and timing signals. It is built into the physical circuit permanently.

Components

ComponentRole
Step CounterTracks which of the 5 execution steps is currently active (T1–T5)
Instruction DecoderReads OP-code from IR, activates corresponding INS signal
Control Signal GeneratorCombinational circuit that produces control signals based on INS + T signals

How Control Signals Depend On:

  • Contents of the step counter (which step T1–T5)
  • Contents of the IR (which instruction)
  • Result of computation (for branch conditions)
  • External signals (interrupt requests)
✅ Advantage: Very FAST — signals generated directly by circuits
⚠️ Disadvantage: Inflexible — hard to modify after manufacturing

What is Microprogrammed Control? ↗ GeeksforGeeks

Instead of hardware circuits, control signals are determined by a program stored in a special memory called the Control Store (microprogram memory). It's a "software" approach to control.

Key Terms

TermMeaning
MicroinstructionAn n-bit word where each bit controls one signal for one step
Micro routineSequence of microinstructions that implements one machine instruction
Control StoreSmall, fast memory on the processor chip that stores microinstructions
μPCMicroprogram counter — tracks which microinstruction to fetch next

How it Works

IR ──► Microinstruction Address Generator ↓ μPC Control Store (microprogram memory) ↓ Microinstructions ↓ Control Signals → hardware components
FeatureHardwiredMicroprogrammed
SpeedFaster ✅Slower
FlexibilityRigid ❌Flexible ✅
ImplementationComplex circuitsSimple, stored program
Used inRISC processorsCISC processors
Modern trend: Since logic circuitry cost has dropped, Hardwired control is now preferred even for complex processors.

Correlation Coefficient r

Table columns: x, y, x², y², xy r = [nΣxy - ΣxΣy] / √{[nΣx²-(Σx)²][nΣy²-(Σy)²]}

Regression Lines

y on x: y - ȳ = byx(x - x̄) where byx = [nΣxy-ΣxΣy] / [nΣx²-(Σx)²] x on y: x - x̄ = bxy(y - ȳ) where bxy = [nΣxy-ΣxΣy] / [nΣy²-(Σy)²] x̄ = Σx/n, ȳ = Σy/n r² = byx × bxy → r = √(byx × bxy)

Rank Correlation (Spearman's)

r = 1 - [6·Σd²] / [n(n²-1)] d = R1 - R2 (difference between ranks) Tied ranks: give average rank, add (m³-m)/12 to Σd² for each tie
💡 For tied ranks: if two values share rank 3 & 4, both get rank 3.5. Add correction (m³-m)/12 where m = number of tied values!

Gradient (∇ϕ)

∇ϕ = (∂ϕ/∂x)i + (∂ϕ/∂y)j + (∂ϕ/∂z)k

Directional Derivative

D = ∇ϕ · â where â = a/|a| (unit vector in given direction) Maximum rate of change = |∇ϕ| (magnitude of gradient) Direction of maximum = ∇ϕ itself

Divergence

If F = Pi + Qj + Rk div F = ∇·F = ∂P/∂x + ∂Q/∂y + ∂R/∂z → scalar result

Curl

curl F = ∇×F = det| i j k | | ∂/∂x ∂/∂y ∂/∂z | | P Q R | = (∂R/∂y - ∂Q/∂z)i - (∂R/∂x - ∂P/∂z)j + (∂Q/∂x - ∂P/∂y)k

Irrotational Field & Scalar Potential

Field is IRROTATIONAL if curl F = 0 To find scalar potential ϕ (where F = ∇ϕ): Step 1: ∂ϕ/∂x = P → integrate w.r.t x → ϕ = ∫P dx + f(y,z) Step 2: ∂ϕ/∂y = Q → differentiate ϕ, compare → find f(y,z) Step 3: ∂ϕ/∂z = R → differentiate ϕ, compare → find complete ϕ

Work Done (Conservative Field)

Work done = ϕ(B) - ϕ(A) where A = initial point, B = final point

Bayes' Theorem

P(Aᵢ|B) = P(Aᵢ) · P(B|Aᵢ) / Σ[P(Aⱼ) · P(B|Aⱼ)] Steps: 1. Make table: Source | P(Aᵢ) | P(B|Aᵢ) | Product 2. Total = sum of all products 3. Answer = product of target / total

Continuous Random Variable

Find constant: ∫f(x)dx = 1 over given range Probability: P(a<X<b) = ∫ₐᵇ f(x)dx Expected value: E(X) = ∫x·f(x)dx Variance: Var(X) = E(X²) - [E(X)]² where E(X²) = ∫x²·f(x)dx

Discrete Random Variable

Find constant: Σf(x) = 1 Expected value: E(X) = Σx·f(x) Variance: Var(X) = E(X²) - [E(X)]²

Binomial Distribution B(n,p)

P(X=r) = ⁿCᵣ · pʳ · qⁿ⁻ʳ where q = 1-p, r = 0,1,2,...n Mean = np, Variance = npq Use when: fixed n trials, each success/fail, p constant

Poisson Distribution

P(X=r) = e⁻λ · λʳ / r! where λ = np Mean = Variance = λ Use when: n large, p small, λ = np is moderate
💡 Binomial vs Poisson: If n is small (like 5) → Binomial. If n is large (like 100) and p is small (like 3%) → Poisson with λ=np!
